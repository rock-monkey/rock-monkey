Windows2000

The best OperatingSystem Microsoft ever made
Why?
  • NTFS. Possibly the best filing system ever invented. More flexible security wise than your average Unix filing system. Ridiculously fast. Introduced with WindowsNT but most home users didn't use that.
  • Stable. Hark if you will, Linux users, but every single crash I've had has been due to faulty hardware. Unlike Windows9x, if an application goes wrong, it doesn't tend to bring the whole system down or write into areas it's not meant to
  • Non-fluffy. Unlike WindowsXP where Microsoft try to hide all the really useful stuff away, Windows2000 lays it bare. Hardware, domain and user administration are simple to control. So what if it's a graphical interface? That's not a _bad_ thing (except for the possibility it might encourage users to change things they don't really understand), and most networking stuff can be done on the command line if you _really_ want to (almost everything can be done on the command-line if you really want to - Dan)
  • Hidden command line. True, command lines are incredibly good for things like pattern recognition (esp. regular expressions), but most humans think in terms of _visual_ things. That's why GUIs are so popular. If you absolutely have to use the command prompt, you can, but you're not forced to
  • No Palladium. Unlike WindowsXP, there isn't any DRM stuff in Windows2000, unless you install WindowsMediaPlayer (latest version), or ServicePack3 or higher. Even then, it just appears as a service which you can disable (no Palladium in any version of Windows at all, actually; DRM only exists in WindowsMediaPlayer, which isn't included in new versions of Windows sold in Europe anyway - Dan)
  • Intuitive. Believe it or not, the Windows layout has remained almost constant, not just because it's more widely recognised, but because it has a very gradual learning curve. But there's still no excuse for putting "Shutdown" on the "Start" menu
  • Activemovie. The simplest video player known to Windows. You don't need WindowsMediaPlayer. Just type "mplayer2" into the Run command, and have a DirectShow interface. That's it. It works.
There's still no excuse for the following, but you're not forced to use them:
  • IIS. It's a horrible webserver, but at least in the Professional edition it doesn't come switched on
  • System sounds, animated icons, fading windows etc... Fair enough, practically every OS has them, but except for the sounds, they aren't turned on by default (not sure I've ever seen animated icons in an un-hacked Windows system: clarify? - Dan)
