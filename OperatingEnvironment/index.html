Operating Environment

A term which isn't used much anymore, and leads to bizarre things like lecturers declaring that all of the following are Operating Systems:

Emacs
Java Virtual Machine
Mozilla Firefox
Windows XP

An Operating Environment is a user-interface for an Operating System. The best example of which is Windows 95. It's actually DOS, with a nice happy graphical front-end for the user to work with. The actual Operating System is still DOS (Albeit DOS 7, which only shipped with Windows 95/8)

A command-line is generally not regarded as an OE as you have to have some way of communicating with the computer

Most distributions of Linux are a combinational Operating System / Operating Environment, with your OE being X, and that in turn being controlled by (typically) either Gnome or KDE

OSs which don't have built in graphical OEs:

Windows 95/98 (although it came with one)
DOS (Windows 3.11 was sold as an OE)
*nix (although it usually comes with one)
MacOS X (although it comes with one)
TOS (The AtariSt operating system. Although it came with GEM on the TOS-chip, it was possible to boot without using GEM to save a few extra Kb of memory)

There are very few OSs with a built in graphical OE. A few that spring to mind are:

Mac Os (Prior to ten)
AmigaOs (Even though the default startup was a CLI, the command 'loadwb' did not start a seperate program, it merely invoked the menu/icon part of the GUI, since the mouse and windows were already running)
WindowsXP (And WindowsNT) (Load one of these in "command line" mode, and it still invokes the GUI, albeit in a cut-down form)

Of course, you could argue that they are 'graphical sub-systems', but then you're just being picky and would probably say that disk-access wasn't part of an Operating System. Loser.
