Measures Implemented

The following technical measures have been implemented to help to prevent Wiki Spam from occuring on RockMonkey:
  • 3rd March 2005: Historical versions of pages now carry a 'robots - noindex/nofollow' tag. Wiki Spam remains profitable even if a wiki is well-maintained and tidied, because the old versions of the pages (with the spam) are still accessibile by search engines. Now, search engines have been asked to ignore these old pages. This change will take several weeks to start having an effect.
  • 3rd March 2005: Added a spam blocker which tests for the sudden addition of a large number of external links to a Wiki Page, and prevents it. If this trips anybody over making a legitimate change, they should leave a note on this page. This spam blocker would prevent all of the Wiki Spam we've seen on RockMonkey to date.

    Pages Affected

    None at present, but is becoming an increasing problem.

    Wiki Spam Cleaning

    How to help in the fight against spam on our wiki:
  • Check the Recent Changes page regularly, and if you find spam, act!
  • Wiki Spam typically looks like a long series of off-site word links, usually at the bottom of a page, but can take other forms, too.
  • Change the page immediately, removing the Wiki Spam. Leave a note on your edit (in the small text box at the bottom of the edit page) stating that you "removed Wiki Spam" (this means that people using the Recent Changes list won't feel the need to look at a page that hasn't actually been changed, and helps us keep track of how bad the problem becomes).
  • If you find Wiki Spam that for some reason you can't change (no time, wierd permissions, etc.), edit this page and put the name of the affected page in the Pages Affected list, above.
