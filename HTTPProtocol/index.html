Simplicity

Client: Hey there, server. I'd like a particular page.
Client: I'm a copy of MozillaFirefox, by the way, and I found you from a link from Google.
Client: I might want something else in a moment, so hang on.
Server: Okay then, here's the page you wanted <hands>.
Client: The page has an image on it, has the image changed since I saw it last week?
Server: Nope, same one since last year, you don't need a new copy.
Client: How about this Java Applet; that's new, right?
Server: Yup, here it is. <hands>
...
Server: Hmm... guess that client has gone away elsewhere, now. Who's next?

Denial of Service

Everyone's heard of Denial of Service attacks. And the "Simplicity" example shows how a basic one can be achieved. See the "...", while the server waits? Yes. You don't tell it you're done, it waits for a defined length of time. The problem is the server can only deal with so many requests at once, so if you keep it waiting by making zillions of requests at once, genuine ones can't get through. Of course, if it was made so the server drops the connection ASAP that would go some way towards fixing it, but then it becomes really inefficient because you have to go through the first 3 lines every single time you do anything...
