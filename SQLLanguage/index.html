SQLLanguage

A bad acronym. Like "ATM Machine"
Structured Query Language is (usually) used to communicate with relational databases like MicrosoftAccess, mySQL and Oracle. It reads almost like plain-English, which was largely the idea. Was designed to be a standard, but then MicrosoftSQL came along and ruined it with commands like "INNER JOIN", which no longer follow the simple metaphors like "SELECT", "UPDATE" and "DELETE"
Is not TuringComplete. Is not terribly flexible for complex requests. But is easy to understand and hugely popular
